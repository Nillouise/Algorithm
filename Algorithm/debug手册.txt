1：写错变量名字，没有考虑题目中的负号，抄模板都抄错
2：忘记初始化全局变量，或者在递归中用一个全局变量，没有考虑子递归改变了全局变量。
3：没有先脑内验算一遍简单样例
4：低估\高估复杂度
5: 没有检查一些不熟悉的地方，如，以为求一个距离r的点是(r/2,r/2),但其实是(r/根号2,r/根号2)
6：使用map，或许x,y之类的键值对时，写成了key，key，或y,y的形式
7: 复杂条件下的边界错误，越界了,或者算的部分不对（从前到后，又从后到前）,复杂的下标一定要在脑内模拟再模拟！！
8: 无限递归导致MLE
9：用了一个数组记录顺序的程序，要特别特别小心。
10： 某些条件下要特判、如满，空之类
11: 有MOD的情况下要在各种算式下写MOD，包括返回值，接收值, 最后输出值
12: 注意，有头尾两端的话，特例也要想一想头尾两端
13: 超时：可能是memset要重置的数组过大。
14: tri树，要算请数组大小，是26*26*26*。。。
15: 爆数据大小，注意用long long，long long不够用就用typedef __int128 LL
16: 打印double时，注意用%.4f而不是%.4lf会更好

dev C++ 设置指南
dev C++ 在Compiler Options的general加上 -std=c++11 ,启用c++11
dev C++ 打开的控制台无法黏贴字符串，要在控制台里右键上面的标题栏，选属性，然后选中快速编辑模式，然后就可以右键黏贴字符串了。
dev C++ 打开调试信息选项 在编译选项--代码生成/优化--连接器--产生调试信息 选yes 即可解决问题
dev C++ 崩溃了，就直接kill 进程

时间策略：
前一个半小时做完最水的水题后，接下来的任何时间里都必须知道3题的题意，再决定要做哪一题。
最后一个半小时时，就专心于某一题，不再换题目了。
总之要注意时间
